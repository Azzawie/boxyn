# Boxyn ‚Äî Design Spec
**Date:** 2026-04-22
**Status:** Approved

## Overview

Boxyn is a smart storage management API built with Rails 8.1. It helps users organize, track, and locate stored items across physical boxes. Users pack items into boxes, photograph them, and each box receives a unique QR code for instant lookup. A full-text search makes any item findable in seconds.

The API is consumed by both a mobile app (iOS/Android) and a web SPA.

---

## Data Model

### User
- `email`, `encrypted_password` (Devise)
- `provider`, `uid` (OAuth ‚Äî Google, Apple)
- Auto-creates a personal Space named "Personal" on signup (role: owner)

### Space
- `name`, `description`
- Logical container for boxes ‚Äî can be personal or shared (home, office, warehouse)
- `has_many :space_memberships`
- `has_many :boxes`

### SpaceMembership
- `user_id`, `space_id`
- `role` ‚Äî enum: `owner | admin | member`
- Roles:
  - **owner** ‚Äî full control, can delete space and manage all members
  - **admin** ‚Äî manage boxes/items, invite members
  - **member** ‚Äî read/write boxes and items, no member management

### Box
- `space_id`, `name`, `description`
- `qr_token` ‚Äî unique UUID, generated on creation, used in QR URL
- `has_many :items`
- `has_one_attached :qr_code_image` ‚Äî PNG generated via rqrcode, stored via Active Storage

### Item
- `box_id`, `name`, `description`
- `search_vector` ‚Äî PostgreSQL `tsvector` column, updated by DB trigger on name/description change
- `has_many :taggings ‚Üí has_many :tags, through: :taggings`
- `has_many_attached :photos`

### Tag
- `name`, `space_id` ‚Äî tags are scoped to a Space for reuse across boxes

### Tagging
- `item_id`, `tag_id` ‚Äî join table

### JwtDenylist
- `jti`, `exp` ‚Äî Devise JWT token revocation on sign-out

## Authentication & Authorization

**Stack:** `devise` + `devise-jwt` + `omniauth-google-oauth2` + `omniauth-apple`

**Token format:** JWT in `Authorization: Bearer <token>` header. Works for mobile and web SPA.

**Auth endpoints:**
```
POST /auth/sign_up
POST /auth/sign_in         ‚Üí returns JWT
POST /auth/sign_out        ‚Üí revokes JWT via JwtDenylist
GET  /auth/:provider/callback  ‚Üí OAuth callback, returns JWT
```

**Authorization:** Enforced via a `SpaceAuthorization` concern included in controllers. Checks `SpaceMembership` role before every action. No external authorization gem ‚Äî logic is simple and inline.

## API Structure

All endpoints under `/api/v1/`. All require a valid JWT.

```
# Spaces
GET    /api/v1/spaces
POST   /api/v1/spaces
PATCH  /api/v1/spaces/:id
DELETE /api/v1/spaces/:id
POST   /api/v1/spaces/:id/memberships       (invite member)
DELETE /api/v1/spaces/:id/memberships/:id   (remove member)

# Boxes
GET    /api/v1/spaces/:space_id/boxes
POST   /api/v1/spaces/:space_id/boxes
GET    /api/v1/boxes/:id                    (full box with items + tags)
PATCH  /api/v1/boxes/:id
DELETE /api/v1/boxes/:id

# QR scan ‚Äî must be declared BEFORE /boxes/:id in routes.rb to avoid conflict
GET    /api/v1/boxes/scan/:qr_token         (requires JWT, returns box + items)

# Items
POST   /api/v1/boxes/:box_id/items
PATCH  /api/v1/items/:id
DELETE /api/v1/items/:id

# Tags
GET    /api/v1/spaces/:space_id/tags
POST   /api/v1/spaces/:space_id/tags

# Search
GET    /api/v1/search?q=keyword&space_id=   (returns items with their box info)
```

Responses serialized with **Blueprinter**. JSON only.

## QR Codes

- `qr_token` (UUID) generated on box creation
- QR encodes: `#{ENV['APP_BASE_URL']}/scan/:qr_token` ‚Äî base URL configured via `APP_BASE_URL` env var
- QR PNG generated by `GenerateQrCodeJob` (Solid Queue) immediately after box creation
- Attached to box via Active Storage (`qr_code_image`)
- Scan endpoint (`GET /api/v1/boxes/scan/:qr_token`) requires valid JWT ‚Äî no public access

---

## Photos & File Handling

- Items use `has_many_attached :photos` via Active Storage
- Upload flow uses **direct uploads** to keep large files off the Rails process:
  1. Client requests signed upload URL ‚Üí `POST /rails/active_storage/direct_uploads`
  2. Client uploads directly to storage backend
  3. Client sends signed blob ID to item endpoint to attach
- Storage backend: **disk** for now, **S3-compatible** in production (config change only, no code change)
- Thumbnails generated via `image_processing` on first request, cached by Active Storage

---

## Search

- PostgreSQL full-text search on `items.search_vector` (`tsvector`)
- Covers `name` + `description` fields
- DB trigger keeps `search_vector` updated automatically on insert/update
- Search endpoint filters by `space_id` to enforce authorization
- Returns: item name, description, tags, box name, box ID ‚Äî so users know exactly where to find the item

---

## Background Jobs (Solid Queue)

`sidekiq` is removed from the Gemfile. Solid Queue handles all async work via PostgreSQL ‚Äî no Redis needed.

**Jobs:**
- `GenerateQrCodeJob` ‚Äî triggered after box creation; generates QR PNG via `rqrcode`, attaches to box via Active Storage
- No other jobs needed at launch; image variants are handled lazily by Active Storage

## Out of Scope (v1)

- Push notifications
- Box location tracking (GPS)
- Public/shareable box links (QR always requires login)
- Bulk item import (CSV)
- Barcode scanning for individual items
